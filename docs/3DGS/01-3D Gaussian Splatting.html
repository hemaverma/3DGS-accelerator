Concept 01

The Problem: Reconstructing 3D from Photos

You have a set of 2D photos taken around an object. You want to synthesize any new viewpoint — not just the ones you captured. Two fundamentally different philosophies emerged to solve this.

NeRF vs 3DGS — Fundamental Approach
NeRF

Neural Radiance Field

Scene = black box neural net.
Shoot rays, query network
at thousands of points,
accumulate color + density.

VS
3DGS

3D Gaussian Splatting

Scene = explicit fuzzy blobs.
Place millions of Gaussians,
project them onto screen,
blend front to back.

The Soap Bubble Analogy

Each blob = one 3D Gaussian. Together they approximate the scene surface and appearance.

Concept 02

What Exactly Is a "Gaussian Blob"?

Each blob is a 3D ellipsoid — like a stretched, rotated egg. It has 5 key properties that get learned during training.

Anatomy of a Single Gaussian

An ellipsoidal Gaussian with
position, shape, color, and opacity

Position (μ)

Where in 3D space is the center of this blob? A single XYZ coordinate.

Shape / Covariance (Σ)

How fat, flat, or stretched is it? Can be a tiny sphere, a thin disk, or a long needle. Encoded as rotation + scale.

Opacity (α)

How transparent is this blob? 0 = invisible ghost, 1 = fully solid.

Color (SH coefficients)

What color does it appear? Stored as spherical harmonics so the color can change depending on view angle — capturing shiny/reflective surfaces.

Count

A typical scene uses 1–6 million Gaussians. Each is independently optimized.

Concept 03

The "Splatting" Step — 3D → 2D

To render a view, each 3D Gaussian is projected ("splatted") onto the 2D image plane. The 3D ellipsoid becomes a 2D ellipse. Then all ellipses are alpha-composited — like stacking transparent stickers, front to back.

Projection: 3D Ellipsoid → 2D Ellipse → Pixel Color

Camera projects 3D Gaussians → 2D splats are depth-sorted and alpha-blended

Alpha Compositing — Blending Overlapping Splats

Front-to-back ordering: each splat contributes proportional to its opacity and remaining "transmittance"

Concept 04

How It Learns — The Training Loop

3DGS starts from a sparse point cloud (from COLMAP structure-from-motion), seeds Gaussians, then iteratively refines them against training photos.

Training Pipeline
📷
Input Photos

Multi-view images + camera poses from COLMAP

✦
Init Points

Sparse point cloud → one Gaussian per point

🎥
Rasterize

Project + splat Gaussians to render a view

⚡
Loss

Compare render vs ground truth photo (L1 + SSIM)

∇
Backprop

Gradients flow back to all Gaussian params

🌱
Densify

Clone/split/prune Gaussians every ~100 steps

Adaptive Density Control — Clone, Split, Prune
Clone

Small Gaussian in under-covered area? Duplicate it and move copy toward gradient.

Split

Large Gaussian covering too much detail? Split into two smaller ones.

Prune

Near-zero opacity Gaussian? Delete it. Keeps scene lean.
