Section 01

System Architecture

3DGS is a three-layer system. The Python training loop orchestrates everything. The C++/CUDA or C++/Metal rasterizer does the heavy lifting. PyTorch's autograd connects them for gradient flow.

System Stack — Top to Bottom

Python is the orchestrator · C++/CUDA or Metal handles the render · autograd bridges the gradient path

NVIDIA CUDA Path
CUDA 11.8+cuBLASThrust

The original implementation. Uses custom CUDA kernels for forward/backward rasterization, Thrust for radix sort, and cuBLAS for matrix ops. Runs on any NVIDIA GPU with compute capability ≥ 7.0 (Volta+).

✓
All RTX cards, A100, H100, datacenter GPUs
✓
Mature toolchain — nvcc, nsight, nvidia-smi
✓
Most research papers target CUDA first
Apple Metal Path (M3)
Metal 3MPSMPSGraph

The M3 GPU has 40 cores and shares memory with the CPU — no PCIe bottleneck. The Metal path uses Metal Performance Shaders (MPS) for PyTorch ops and custom Metal compute kernels for rasterization.

✓
M1/M2/M3 — unified memory, zero copy
✓
PyTorch MPS backend (torch.device("mps"))
~
Community ports maturing — gsplat is best option
Section 02

Core Data Structures

Everything lives in Structure of Arrays (SoA) format — not Array of Structures. This is critical for GPU memory coalescing. Each property of all 3M Gaussians is stored in its own contiguous tensor.

GaussianModel — The Scene Representation (Python / PyTorch)
Python · gaussian_model.py
import torch import torch.nn as nn class GaussianModel: def __init__(self, sh_degree: int = 3): # ── Core parameters (59 floats per Gaussian) ────────────────── self._xyz = None # [N, 3] — position μ self._scaling = None # [N, 3] — log scale (exp to get sx,sy,sz) self._rotation = None # [N, 4] — quaternion (w,x,y,z) self._opacity = None # [N, 1] — logit opacity (sigmoid to get α) self._features_dc = None # [N, 1, 3] — DC SH color (degree 0) self._features_rest= None # [N, 15, 3] — higher-degree SH (degrees 1–3) # ── Optimizer state ─────────────────────────────────────────── self.optimizer = None self.max_sh_degree = sh_degree self.active_sh_degree = 0 # ramp up during training def create_from_pcd(self, pcd_points: torch.Tensor, device: str): # Detect device — works for both CUDA and Metal MPS dev = torch.device(device) # "cuda:0" or "mps" N = pcd_points.shape[0] self._xyz = nn.Parameter(pcd_points.to(dev).requires_grad_(True)) # Initialise scales from nearest-neighbour distances dists = compute_nn_distances(pcd_points) scales = torch.log(dists.unsqueeze(-1).repeat(1,3)) self._scaling = nn.Parameter(scales.to(dev)) # Identity rotation (w=1, x=y=z=0) rots = torch.zeros(N, 4, device=dev) rots[:, 0] = 1.0 self._rotation = nn.Parameter(rots) # Logit(0.1) ≈ -2.2 — start near-transparent self._opacity = nn.Parameter( torch.full((N, 1), -2.2, device=dev)) @property def get_xyz(self) -> torch.Tensor: return self._xyz @property def get_scaling(self) -> torch.Tensor: return torch.exp(self._scaling) @property def get_rotation(self)-> torch.Tensor: return torch.nn.functional.normalize(self._rotation, dim=-1) @property def get_opacity(self) -> torch.Tensor: return torch.sigmoid(self._opacity)
Why SoA beats AoS on GPU

SoA (Structure of Arrays) lets GPU warps load the same field from consecutive memory addresses — maximising memory bandwidth utilisation

Section 03

CUDA Rasterizer Kernels

The rasterizer is two CUDA kernels: preprocess (project + sort) and render (alpha composite). The backward pass has matching kernels that compute gradients w.r.t. all Gaussian parameters.

Kernel 1 of 2 — Preprocess: Project 3D Gaussians to 2D
CUDA C++ · preprocess.cu
// Each CUDA thread handles ONE Gaussian // gridDim × blockDim must cover all N Gaussians __global__ void preprocessCUDA( int P, // number of Gaussians const float* positions, // [P,3] world-space centres const float* scales, // [P,3] scale sx,sy,sz const float* rotations, // [P,4] quaternion w,x,y,z const float* opacities, // [P,1] pre-sigmoid opacity const float* shs, // [P,48] spherical harmonic coeffs const float* view_matrix, // [4,4] world-to-camera const float* proj_matrix, // [4,4] camera-to-clip // outputs float2* xy_screen, // [P,2] 2D screen position float4* cov2D, // [P,3] upper-tri of 2D cov (a,b,c) float* depths, // [P] depth for sorting uint32_t*tiles_touched, // [P] how many 16×16 tiles overlap float3* rgb // [P,3] SH-evaluated colour ) { int idx = blockIdx.x * blockDim.x + threadIdx.x; if (idx >= P) return; // guard — last block may be partial // 1. Load this Gaussian's position and transform to camera space float3 p = {positions[idx*3], positions[idx*3+1], positions[idx*3+2]}; float4 p_cam = transformPoint4x4(p, view_matrix); depths[idx] = p_cam.z; // store depth for radix sort // 2. Build 3D covariance Σ = R·S·Sᵀ·Rᵀ float3 s = {scales[idx*3], scales[idx*3+1], scales[idx*3+2]}; float4 q = {rotations[idx*4], rotations[idx*4+1], rotations[idx*4+2], rotations[idx*4+3]}; float3x3 Sigma3D = computeCov3D(s, q); // builds R·S·Sᵀ·Rᵀ // 3. Project: Σ' = J·W·Σ·Wᵀ·Jᵀ (EWA approximation) float3 cov = computeCov2D(p, Sigma3D, view_matrix, proj_matrix); // cov = (a, b, c) upper triangle of 2×2 cov matrix // [a b] a = cov.x, b = cov.y, c = cov.z // [b c] // 4. Add low-pass filter (prevents aliasing at training res) cov.x += 0.3f; cov.z += 0.3f; // 5. Project to screen space float2 screen_pos = ndc2Pix(p_cam, /*width, height*/); xy_screen[idx] = screen_pos; cov2D[idx] = {cov.x, cov.y, cov.z, 0.0f}; // 6. Count which 16×16 screen tiles this Gaussian overlaps // (used to distribute work in the render kernel) float radius = computeRadius(cov); // 3σ in pixels tiles_touched[idx] = countTiles(screen_pos, radius); // 7. Evaluate spherical harmonics for view-dependent colour float3 dir = normalize(p - getCameraPos(view_matrix)); rgb[idx] = evalSH(shs + idx*48, dir, /*degree=*/3); }
Kernel 2 of 2 — Render: Tile-Based Alpha Compositing
CUDA C++ · forward.cu — the critical inner loop
// One CUDA block = one 16×16 tile of pixels // Threads within a block cooperate via shared memory __global__ void renderCUDA( const uint2* ranges, // [tiles] range of sorted Gaussian indices for this tile const uint32_t*point_list, // sorted Gaussian indices (back-to-front within tile) const float2* points_xy, // [P,2] screen positions const float4* cov2D, // [P,3] 2D covariance (a,b,c) const float* opacities, // [P] per-gaussian opacity const float3* colours, // [P,3] SH-evaluated RGB float* out_colour // [H,W,3] output image ) { // Identify which tile + which pixel within the tile uint2 tile = {blockIdx.x, blockIdx.y}; uint2 pixel = {tile.x*16 + threadIdx.x, tile.y*16 + threadIdx.y}; // Shared memory — all 256 threads cooperate to load Gaussian data __shared__ float2 collected_xy[256]; __shared__ float4 collected_cov[256]; __shared__ float collected_alpha[256]; __shared__ float3 collected_colour[256]; // Accumulation state per pixel float3 C = {0, 0, 0}; // accumulated colour float T = 1.0f; // transmittance (starts full) const float T_THRESH = 0.0001f; // early-exit when nearly opaque // Get range of Gaussians for this tile (already depth-sorted) uint2 range = ranges[tile.y * grid_w + tile.x]; int rounds = (range.y - range.x + 255) / 256; // batches of 256 for (int r = 0; r < rounds; ++r) { // ── Cooperative load: each thread loads one Gaussian ──────── int g_idx = range.x + r*256 + threadIdx.x * 16 + threadIdx.y; if (g_idx < range.y) { int g = point_list[g_idx]; collected_xy [threadIdx.x*16+threadIdx.y] = points_xy[g]; collected_cov [threadIdx.x*16+threadIdx.y] = cov2D[g]; collected_alpha [threadIdx.x*16+threadIdx.y] = opacities[g]; collected_colour[threadIdx.x*16+threadIdx.y] = colours[g]; } __syncthreads(); // wait — all 256 threads must finish loading // ── Each thread processes all 256 loaded Gaussians ─────────── for (int j = 0; j < min(256, range.y-range.x-r*256); ++j) { if (T < T_THRESH) break; // early exit — pixel fully opaque // Evaluate 2D Gaussian at this pixel position float2 d = {pixel.x - collected_xy[j].x, pixel.y - collected_xy[j].y}; float4 cov = collected_cov[j]; // Mahalanobis distance: dᵀ Σ⁻¹ d float det = cov.x*cov.z - cov.y*cov.y; float power = -0.5f * (cov.z*d.x*d.x - 2*cov.y*d.x*d.y + cov.x*d.y*d.y) / det; if (power > 0.0f) continue; // outside Gaussian // α = opacity × G(pixel) where G is the 2D Gaussian value float alpha = min(0.99f, collected_alpha[j] * expf(power)); if (alpha < 1.0f/255.0f) continue; // negligible contribution // Accumulate: C += c_i * α_i * T_i C.x += collected_colour[j].x * alpha * T; C.y += collected_colour[j].y * alpha * T; C.z += collected_colour[j].z * alpha * T; T *= (1.0f - alpha); // reduce remaining transmittance } __syncthreads(); // sync before next batch } // Write final pixel colour if (pixel.x < width && pixel.y < height) { int pix_id = pixel.y * width + pixel.x; out_colour[pix_id*3 ] = C.x + T * bg_r; // background fill out_colour[pix_id*3+1] = C.y + T * bg_g; out_colour[pix_id*3+2] = C.z + T * bg_b; } }
Why Tile-Based? — Shared Memory Diagram

Screen divided into 16×16 tiles · Each CUDA block handles one tile · 256 threads cooperate through shared memory to load Gaussians in batches — avoids redundant global memory reads

Section 04

Metal Compute Kernels (M3 Apple Silicon)

The Metal path mirrors the CUDA architecture but uses Metal Shading Language (MSL). The M3's unified memory architecture eliminates CPU↔GPU copies — tensors live in a single address space accessible by both.

On M3, torch.device("mps") is the entry point. For the rasterizer itself, the best production-ready option is gsplat which has a Metal-compatible backend via MPS. The kernels below show the manual Metal path for learning/custom implementations.
Metal Preprocess Kernel — Gaussian Projection (MSL)
Metal Shading Language · preprocess.metal
#include using namespace metal; // Matches CUDA __device__ — runs on each GPU thread // thread_position_in_grid = threadIdx in CUDA terms kernel void preprocessGaussians( // Buffers — Metal's equivalent of CUDA device pointers const device float3* positions [[ buffer(0) ]], const device float3* scales [[ buffer(1) ]], const device float4* rotations [[ buffer(2) ]], // quaternion const device float* opacities [[ buffer(3) ]], const device float4x4*view_matrix[[ buffer(4) ]], const device float4x4*proj_matrix[[ buffer(5) ]], device float2* xy_screen [[ buffer(6) ]], device float3* cov2D [[ buffer(7) ]], // a,b,c device float* depths [[ buffer(8) ]], // Grid dispatch uint idx [[ thread_position_in_grid ]] // = blockIdx*blockDim+threadIdx ) { // 1. Transform position to camera space float4 pos_h = float4(positions[idx], 1.0f); float4 p_cam = view_matrix[0] * pos_h; // metal: col-major mul depths[idx] = p_cam.z; // 2. Build 3D covariance Σ = R·S·Sᵀ·Rᵀ from quaternion + scale float4 q = rotations[idx]; float3 s = scales[idx]; // Quaternion → rotation matrix float3x3 R = float3x3( 1-2*(q.z*q.z+q.w*q.w), 2*(q.y*q.z-q.x*q.w), 2*(q.y*q.w+q.x*q.z), 2*(q.y*q.z+q.x*q.w), 1-2*(q.y*q.y+q.w*q.w), 2*(q.z*q.w-q.x*q.y), 2*(q.y*q.w-q.x*q.z), 2*(q.z*q.w+q.x*q.y), 1-2*(q.y*q.y+q.z*q.z) ); // S = diag(sx², sy², sz²) → Σ = R · S · Rᵀ float3x3 Sigma = R * float3x3(s.x*s.x,0,0, 0,s.y*s.y,0, 0,0,s.z*s.z) * transpose(R); // 3. EWA projection: Σ' = J·W·Σ·Wᵀ·Jᵀ float3x3 W = float3x3(view_matrix[0]); // upper-left 3×3 of view float3 t = p_cam.xyz; // Jacobian of perspective divide float3x3 J = float3x3( 1/t.z, 0, -t.x/(t.z*t.z), 0, 1/t.z, -t.y/(t.z*t.z), 0, 0, 0 ); float3x3 T = J * W; float3x3 Sigma2 = T * Sigma * transpose(T); // Store upper-triangle (a,b,c) + low-pass filter cov2D[idx] = float3(Sigma2[0][0]+0.3f, Sigma2[0][1], Sigma2[1][1]+0.3f); // 4. Project to NDC then screen pixels float4 p_ndc = proj_matrix[0] * p_cam; xy_screen[idx] = float2( (p_ndc.x / p_ndc.w + 1.0f) * 0.5f * float(W_px), (1.0f - p_ndc.y / p_ndc.w) * 0.5f * float(H_px) ); }
Metal Render Kernel — Alpha Compositing (MSL)
Metal Shading Language · render.metal
// Metal equivalent of the CUDA renderCUDA kernel // threadgroup = CUDA __shared__ | threads_per_threadgroup = blockDim kernel void renderGaussians( const device uint2* tile_ranges [[ buffer(0) ]], const device uint* sorted_ids [[ buffer(1) ]], const device float2* xy [[ buffer(2) ]], const device float3* cov2D [[ buffer(3) ]], const device float* opacities [[ buffer(4) ]], const device float3* colours [[ buffer(5) ]], device float4* out_image [[ buffer(6) ]], // RGBA // Threadgroup shared memory — declared here, not inside the function // Metal requires known size at compile time threadgroup float2* s_xy [[ threadgroup(0) ]], threadgroup float3* s_cov [[ threadgroup(1) ]], threadgroup float* s_alpha [[ threadgroup(2) ]], threadgroup float3* s_col [[ threadgroup(3) ]], // Thread IDs uint2 gid [[ thread_position_in_grid ]], // global pixel coord uint2 lid [[ thread_position_in_threadgroup ]], // local (0..15) uint2 tgid [[ threadgroup_position_in_grid ]] // tile coord ) { uint tg_flat = lid.y * 16 + lid.x; // 0..255 within tile uint2 range = tile_ranges[tgid.y * grid_w + tgid.x]; float3 C = float3(0); float T = 1.0f; for (uint r = 0; r * 256 < range.y - range.x; ++r) { // Cooperative load from device memory into threadgroup memory uint g_load = range.x + r*256 + tg_flat; if (g_load < range.y) { uint g = sorted_ids[g_load]; s_xy [tg_flat] = xy[g]; s_cov [tg_flat] = cov2D[g]; s_alpha[tg_flat] = opacities[g]; s_col [tg_flat] = colours[g]; } threadgroup_barrier(mem_flags::mem_threadgroup); // = __syncthreads() for (uint j = 0; j < 256 && T > 1e-4f; ++j) { float2 d = float2(gid) - s_xy[j]; float3 cov = s_cov[j]; float det = cov.x*cov.z - cov.y*cov.y; float power = -0.5f*(cov.z*d.x*d.x -2*cov.y*d.x*d.y +cov.x*d.y*d.y) / det; if (power > 0.0f) continue; float alpha = min(0.99f, s_alpha[j] * exp(power)); if (alpha < 1.0f/255.0f) continue; C += s_col[j] * alpha * T; T *= (1.0f - alpha); } threadgroup_barrier(mem_flags::mem_threadgroup); } if (gid.x < width && gid.y < height) out_image[gid.y*width+gid.x] = float4(C + T*bg_col, 1.0f); }

import Metal, Foundation class GaussianRasterizer { let device: MTLDevice let queue: MTLCommandQueue let library: MTLLibrary var preprocessPSO: MTLComputePipelineState! var renderPSO: MTLComputePipelineState! init() { device = MTLCreateSystemDefaultDevice()! // M3 GPU queue = device.makeCommandQueue()! // Compile .metal files at build time → embedded in app bundle library = device.makeDefaultLibrary()! let preFn = library.makeFunction(name: "preprocessGaussians")! let renFn = library.makeFunction(name: "renderGaussians")! preprocessPSO = try! device.makeComputePipelineState(function: preFn) renderPSO = try! device.makeComputePipelineState(function: renFn) } func render(gaussians: GaussianBuffers, camera: CameraParams) { let cmd = queue.makeCommandBuffer()! let enc = cmd.makeComputeCommandEncoder()! // ── Preprocess pass ─────────────────────────────────────────── enc.setComputePipelineState(preprocessPSO) enc.setBuffer(gaussians.positions, offset: 0, index: 0) enc.setBuffer(gaussians.scales, offset: 0, index: 1) enc.setBuffer(gaussians.rotations, offset: 0, index: 2) // ... bind remaining buffers let N = gaussians.count let tSize = MTLSize(width: 256, height: 1, depth: 1) let gSize = MTLSize(width: (N+255)/256, height: 1, depth: 1) enc.dispatchThreadgroups(gSize, threadsPerThreadgroup: tSize) // ── Sort by depth (MPS radix sort) ──────────────────────────── // MPSMatrixSortByKey or custom bitonic sort // ── Render pass ─────────────────────────────────────────────── enc.setComputePipelineState(renderPSO) let tileGrid = MTLSize(width: (W+15)/16, height: (H+15)/16, depth: 1) let tileSize = MTLSize(width: 16, height: 16, depth: 1) enc.dispatchThreadgroups(tileGrid, threadsPerThreadgroup: tileSize) enc.endEncoding() cmd.commit() // async — GPU runs while CPU continues } }
Section 05

Python Training Loop — Device-Agnostic

The Python layer works for both CUDA and Metal (MPS) by using PyTorch's device abstraction. Only the rasterizer call differs — everything else is identical.

Training Loop — Works on CUDA and MPS
Python · train.py
import torch from gsplat import rasterization # gsplat: works on CUDA + MPS def train(scene_path: str): # ── Device detection ────────────────────────────────────────── if torch.cuda.is_available(): device = torch.device("cuda:0") print(f"CUDA → {torch.cuda.get_device_name(0)}") elif torch.backends.mps.is_available(): device = torch.device("mps") # M1/M2/M3 GPU print("Metal MPS → Apple Silicon GPU") else: device = torch.device("cpu") # fallback # ── Initialise model ────────────────────────────────────────── gaussians = GaussianModel(sh_degree=3) pcd = load_point_cloud(scene_path) # from COLMAP gaussians.create_from_pcd(pcd, device=str(device)) # ── Optimizer — different LR per parameter group ────────────── opt = torch.optim.Adam([ {'params': [gaussians._xyz], 'lr': 1.6e-4}, {'params': [gaussians._scaling], 'lr': 5e-3 }, {'params': [gaussians._rotation], 'lr': 1e-3 }, {'params': [gaussians._opacity], 'lr': 5e-2 }, {'params': [gaussians._features_dc], 'lr': 2.5e-3}, {'params': [gaussians._features_rest], 'lr': 2.5e-4}, ], eps=1e-15) training_cameras = load_cameras(scene_path) for step in range(30_000): # ── Pick a training view ────────────────────────────────── cam = random.choice(training_cameras) # ── Forward: render from current Gaussians ──────────────── # gsplat.rasterization dispatches the right backend rendered, alphas, meta = rasterization( means = gaussians.get_xyz, # [N,3] quats = gaussians.get_rotation, # [N,4] scales = gaussians.get_scaling, # [N,3] opacities= gaussians.get_opacity, # [N,1] colors = eval_sh(gaussians._features_dc, gaussians._features_rest, cam.direction), # [N,3] viewmats = cam.view_matrix.unsqueeze(0), # [1,4,4] Ks = cam.intrinsics.unsqueeze(0), # [1,3,3] width = cam.width, height=cam.height, ) # ── Loss: L1 + SSIM ────────────────────────────────────── gt = cam.gt_image.to(device) l1 = l1_loss(rendered, gt) ssim_v = ssim_loss(rendered, gt) loss = (1 - 0.2) * l1 + 0.2 * ssim_v # ── Backward + step ────────────────────────────────────── opt.zero_grad() loss.backward() # gradients flow through gsplat → all 59 params opt.step() # ── Densification (every 100 steps) ────────────────────── if step % 100 == 0 and step < 15_000: densify_and_prune(gaussians, opt, meta) # ── Opacity reset (every 3000 steps) ───────────────────── if step % 3000 == 0: gaussians._opacity.data.clamp_(max=inverse_sigmoid(0.01)) # ── Ramp up SH degree every 1000 steps ─────────────────── if step % 1000 == 0: gaussians.active_sh_degree = min( gaussians.active_sh_degree + 1, gaussians.max_sh_degree)
Section 06

Memory Architecture

Memory is the biggest constraint. 3M Gaussians × 59 params × 4 bytes = ~700 MB just for parameters. Add gradients, optimizer state (Adam keeps 2 moments), and intermediate buffers and you're at 3–5 GB for training.

Memory Layout Comparison — CUDA vs M3 Unified Memory

CUDA must copy data over PCIe when moving from CPU — M3 unified memory eliminates this entirely. The same physical DRAM is CPU + GPU memory.

Quick Start — Capture and Train Your Own Scene
bash + Python · end-to-end
# ── 1. Capture: take 100–200 photos around object/scene ──────── # ── 2. Run COLMAP to get camera poses + sparse point cloud ───── colmap automatic_reconstructor \ --workspace_path ./my_scene \ --image_path ./my_scene/images # ── 3. Convert COLMAP output → gsplat format ──────────────────── python convert.py -s ./my_scene # ── 4. Train — auto-detects CUDA or MPS ───────────────────────── python train.py \ --source_path ./my_scene \ --model_path ./output \ --iterations 30000 \ --densify_until_iter 15000 # ── 5. Render novel views ──────────────────────────────────────── python render.py -m ./output # ── M3-specific: limit Gaussians for 8GB unified memory ───────── python train.py --source_path ./my_scene \ --max_num_splats 1_000_000 \ # cap at 1M for 8GB --cap_max_gaussians 1_500_000 # hard ceiling
On M3 with 8GB unified memory, the OS reserves ~2GB, leaving ~6GB. At 3M Gaussians this is insufficient. Use --max_num_splats 800000 for safe training. M3 Pro (18GB) or M3 Max (36/48GB) can run full scenes without limits.
The gsplat library (pip install gsplat) is the recommended implementation for both CUDA and MPS. It's actively maintained, has a clean API, and supports mixed-precision training which halves memory usage — critical for M3 with 8GB.
